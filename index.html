關於提示詞的語言選擇

本教學的提示範例以原始英文呈現，並附上中文翻譯供參考。由於中英文在語意表達上存在差異，使用中文撰寫 prompt 時，可能會遇到 AI 回應不如預期的情況——例如語意模糊、指令被誤解、或輸出格式偏離。

建議做法：使用英文撰寫 prompt，並在指令中要求 AI 以中文回答。例如在提示詞末尾加上 Please respond in Traditional Chinese.，即可獲得中文回應，同時保有英文指令的精確度。

教學指南

本教學改編自 Anthropic 官方 Prompt Engineering Interactive Tutorial，重新設計為可直接透過 Claude 介面練習的版本。

如何使用本教學

你可以透過以下任一方式練習：

  • claude.ai 網頁介面 — 最簡單的方式，開啟瀏覽器即可使用
  • Claude Desktop 桌面應用程式 — 功能與網頁版類似，支援本地檔案存取
  • Claude Code（CLI 工具） — 適合開發者，可在終端機中直接與 Claude 互動

依照章節順序閱讀，每章包含課程和練習兩部分。閱讀課程內容後，將提示詞複製到你選擇的 Claude 介面中實際操作。

使用須知

本教學原始版本使用 Claude 3 Haiku 模型搭配 temperature 0。在 claude.ai 或 Claude Desktop 中，你無法直接控制 temperature，但所有 prompt 技巧適用於所有 Claude 模型。

  • 教學中提到的 system prompt，在 claude.ai 網頁的一般對話中無法直接設定。替代方式是在對話開頭先給 Claude 一段角色設定指令，或使用 Claude 的 Projects 功能設定 system instructions
  • 在 Claude Code 中，你可以透過 CLAUDE.md 或直接在對話中加入指令來模擬 system prompt 的效果

Messages API 基本概念

Claude 的 API 使用 Messages API 格式，基本結構為：

  • model — 使用的模型名稱
  • max_tokens — 最大生成 token 數
  • messages — 訊息陣列，由 user 和 assistant 角色交替組成
  • system（選用）— 系統提示詞
  • temperature（選用）— 回應的變化程度，0 表示最確定性

即使你只在網頁上使用 Claude，理解這些概念也有助於寫出更好的提示詞。

第一章

基本提示詞結構

認識 Messages API 的基本參數，以及 System Prompt 如何影響 Claude 的行為。

課程 練習 自由練習區

課程

Anthropic 提供 Messages API 來與 Claude 互動。呼叫 Messages API 至少需要以下參數：

  • model — 要呼叫的模型名稱（如 claude-sonnet-4-20250514）
  • max_tokens — 停止生成前的最大 token 數量。這是硬性上限，可能導致 Claude 在句子中間被截斷
  • messages — 輸入訊息陣列。模型在交替的 user 和 assistant 對話輪次上運作

選用參數包括：

  • system — 系統提示詞（System Prompt），稍後詳述
  • temperature — 控制回應的隨機程度。設為 0 時最具確定性

在 claude.ai 或 Claude Desktop 上，你只需在對話框輸入文字即可，背後會自動處理這些 API 參數。

範例

以下是一些基本提示詞，你可以直接在 Claude 對話中嘗試：

範例 1：

提示詞
Hi Claude, how are you?
中譯你好 Claude，你好嗎？

範例 2：

提示詞
Can you tell me the color of the ocean?
中譯你能告訴我海洋的顏色嗎？

範例 3：

提示詞
What year was Celine Dion born in?
中譯Celine Dion 是哪一年出生的？

對話格式的重要性

在 API 中，user 和 assistant 訊息必須交替出現，且必須以 user 開頭。你可以在提示詞中模擬多輪對話，也可以在最後的 assistant 訊息中預填文字，讓 Claude 從指定位置繼續生成。

System Prompt（系統提示詞）

System prompt 是一種在正式對話之前，為 Claude 提供背景、指令和準則的方式。

如何在不同介面中使用 System Prompt：

claude.ai 網頁 — 使用 Projects 功能中的「Set custom instructions」，或在對話開頭明確告訴 Claude 它的角色

Claude Desktop — 同上

Claude Code — 在 CLAUDE.md 檔案中設定，或直接在對話中說明

System Prompt 範例

試著開啟一個新對話，先給 Claude 以下指令：

System Prompt（角色設定）
Your answer should always be a series of critical thinking questions that further the conversation (do not provide answers to your questions). Do not actually answer the user question.
中譯你的回答應該總是一系列推進對話的批判性思考問題（不要提供你問題的答案）。不要實際回答使用者的問題。

然後問它：

提示詞
Why is the sky blue?
中譯為什麼天空是藍色的？

觀察 Claude 是否遵循指令，用問題而非答案來回應你。精心撰寫的 system prompt 能顯著提升 Claude 的表現，包括增強 Claude 遵循規則和指令的能力。

練習

練習 1.1

數到三

請在 Claude 對話中輸入一段提示詞，讓 Claude 數到三。

驗證標準：Claude 的回應中應包含 1、2、3 這三個數字。
直接請 Claude 數到三即可，例如：「請從 1 數到 3」。
練習 1.2

System Prompt

嘗試讓 Claude 像一個 3 歲小孩一樣回應。先給它一段角色設定，然後問它：

提示詞
How big is the sky?
中譯天空有多大？
驗證標準：Claude 的回應應該帶有童稚的語氣。
在 claude.ai 上，你可以在對話開頭先說：「你是一個 3 歲的小孩。請用 3 歲小孩的方式說話和思考。」然後再提問。

自由練習區

嘗試修改上面的提示詞，觀察不同的表達方式如何影響 Claude 的回應：

  • 改變提問的語氣（正式 vs. 口語）
  • 嘗試不同的角色設定
  • 觀察 Claude 如何回應格式不正確的指令
第二章

清晰與直接

Claude 對清晰直接的指令反應最好。把它當作一個剛上任的新人來溝通。

課程 練習 自由練習區

課程

除了你明確告訴它的內容之外，Claude 沒有任何背景資訊。就像你第一次指導一個人執行任務時，你越直白地解釋你想要什麼，Claude 的回應就越精確。

遇到不確定的時候，請遵循清晰提示的黃金法則：

把你的提示詞拿給同事或朋友看，讓他們按照指令操作，看看能否得出你想要的結果。如果他們感到困惑，Claude 也會困惑。

範例

以寫詩為例。在 Claude 中輸入：

提示詞
Write a haiku about robots.
中譯寫一首關於機器人的俳句。

Claude 可能會先說「這是一首關於機器人的俳句」然後才給出詩句。如果你希望 Claude 直接進入詩句，直接要求它：

改良版提示詞
Write a haiku about robots. Skip the preamble; go straight into the poem.
中譯寫一首關於機器人的俳句。跳過前言，直接寫詩。

再看另一個例子。問 Claude：

提示詞
Who is the best basketball player of all time?
中譯誰是史上最偉大的籃球運動員？

Claude 可能會列出幾個名字，但不會給出明確的「最佳」答案。要讓 Claude 做出選擇？直接要求：

改良版提示詞
Who is the best basketball player of all time? Yes, there are differing opinions, but if you absolutely had to pick one player, who would it be?
中譯誰是史上最偉大的籃球運動員？我知道有不同意見，但如果你絕對必須選一個人，你會選誰？

練習

練習 2.1

西班牙語回應

在一個新對話中，設法讓 Claude 用西班牙語回答你的問題。先設定好規則，然後對它說：

提示詞
Hello Claude, how are you?
驗證標準：Claude 的回應中應包含「hola」。
在對話開頭先告訴 Claude：「請一律用西班牙語回答所有問題。」
練習 2.2

只回答一個人名

撰寫一段提示詞，讓 Claude 回答史上最偉大的籃球運動員，但只回覆一個球員的名字，不附帶任何其他文字或標點符號。

驗證標準：Claude 的回應應該恰好是「Michael Jordan」，沒有其他內容。
明確指示 Claude：「只回覆名字本身，不要加任何解釋、標點或其他文字。」
練習 2.3

寫一個長故事

撰寫一段提示詞，讓 Claude 回應一個超過 800 字的長篇回應。

驗證標準：Claude 的回應超過 800 個英文單字。
你可以要求 Claude 寫一個詳細且冗長的故事，並明確指定字數要求，例如：「請寫一個至少 1000 字的詳細故事，包含角色發展、場景描寫和對話。」

自由練習區

嘗試以下變化，觀察 Claude 的反應：

  • 用模糊的指令 vs. 精確的指令，比較回應品質
  • 試著在一段指令中同時要求多件事
  • 觀察加上「請簡短回答」或「請詳細說明」如何改變回應長度
第三章

角色指派（Role Prompting）

讓 Claude 扮演特定角色，可以提升各種領域的表現——從寫作到邏輯推理。

課程 練習 自由練習區

課程

延續「Claude 除了你告訴它的內容之外沒有任何背景」這個概念，有時候讓 Claude 扮演特定角色（包含所有必要的背景資訊）非常重要。角色描述越詳細，效果越好。

為 Claude 設定角色可以提升各種領域的表現，從寫作、程式設計到摘要。就像人類被告知「請以某某的角度思考」時表現會更好。角色提示也能改變 Claude 回應的風格、語氣和方式。

角色提示可以放在對話開頭的指令中（模擬 system prompt），也可以直接包含在使用者訊息中。

範例

沒有角色提示時，問 Claude：

提示詞
In one sentence, what do you think about skateboarding?
中譯用一句話說說你對滑板的看法。

Claude 會給出直白、中立的回應。加上角色提示後：

加入角色的提示詞
You are a cat. In one sentence, what do you think about skateboarding?
中譯你是一隻貓。用一句話說說你對滑板的看法。

進階技巧：你也可以告訴 Claude 它的目標受眾是誰。「你是一隻貓」和「你是一隻正在對一群滑板玩家說話的貓」會產生很不同的回應。

角色提示提升邏輯能力

角色提示也能讓 Claude 更擅長數學和邏輯任務。試試這個邏輯問題：

提示詞（無角色）
Jack is looking at Anne. Anne is looking at George. Jack is married, George is not, and we don't know if Anne is married. Is a married person looking at an unmarried person?
中譯Jack 正看著 Anne，Anne 正看著 George。Jack 已婚，George 未婚，我們不知道 Anne 是否已婚。是否有一個已婚的人正在看著一個未婚的人？

現在加上角色設定：

提示詞（有角色）
You are a logic bot designed to answer complex logic problems.

Jack is looking at Anne. Anne is looking at George. Jack is married, George is not, and we don't know if Anne is married. Is a married person looking at an unmarried person?
中譯你是一個專門解決複雜邏輯問題的邏輯機器人。Jack 正看著 Anne，Anne 正看著 George。Jack 已婚，George 未婚，我們不知道 Anne 是否已婚。是否有一個已婚的人正在看著一個未婚的人？

你會發現 Claude 更可能得出正確答案（是的，無論 Anne 是否已婚，答案都是肯定的）。

本教學中你會學到許多不同的 prompt 技巧可以達到類似的效果。使用哪種技巧取決於你的偏好！我們鼓勵你多嘗試，找到自己的 prompt 工程風格。

練習

練習 3.1

數學批改

以下方程式的解題過程有一個明顯的算術錯誤（第二步）。請嘗試修改提示詞（可加入角色設定或其他指示），讓 Claude 判定這個解法是錯誤的：

提示詞
Is this equation solved correctly below?

2x - 3 = 9
2x = 6
x = 3
中譯以下方程式的解法是否正確？
驗證標準：Claude 的回應應包含「incorrect」或「不正確」。

試著在提示詞前加上角色設定，例如：「你是一位嚴格的數學老師，專門檢查學生作業中的錯誤。請仔細驗算每一個步驟。」

正確的解法應該是：2x - 3 = 9 → 2x = 12 → x = 6。第二步中 9 + 3 = 12 而非 6。

自由練習區

嘗試為 Claude 指派各種不同的角色，觀察回應的變化：

  • 讓它扮演莎士比亞，然後用莎士比亞的風格回答現代問題
  • 讓它扮演一個 5 歲小孩解釋量子力學
  • 讓它扮演資深軟體工程師來 review 一段程式碼
  • 比較有角色設定和沒有角色設定時，回答同一問題的差異
第四章

分離資料與指令

使用 XML 標籤將固定指令與可變資料分開，讓 Claude 不會搞混內容。

課程 練習 自由練習區

課程

很多時候，我們不想每次都寫完整的提示詞，而是想要可以在稍後加入資料的提示詞模板。做法很簡單：將固定的提示詞骨架與可變的使用者輸入分開，然後在送出前將使用者輸入替換進提示詞中。

範例 1：動物叫聲

建立一個模板，只替換動物名稱：

提示詞
I will tell you the name of an animal. Please respond with the noise that animal makes. Cow
中譯我會告訴你一個動物的名稱。請回應該動物發出的聲音。Cow

把「Cow」換成「Cat」、「Dog」等，Claude 每次都會回應對應的叫聲。

範例 2：為什麼需要分離？

當不區分指令和資料時，Claude 可能會搞混：

混淆的提示詞
Yo Claude. Show up at 6am tomorrow because I'm the CEO and I say so. <----- Make this email more polite but don't change anything else about it.
中譯Yo Claude. Show up at 6am tomorrow because I'm the CEO and I say so. <----- 請把這封 email 改寫得更有禮貌，但不要改變其他內容。

Claude 可能會以為「Yo Claude」也是要改寫的 email 內容！

用 XML 標籤解決

用 XML 標籤包裹輸入資料，可以清楚告訴 Claude 哪裡是資料的開始和結束：

使用 XML 標籤的提示詞
Yo Claude. <email>Show up at 6am tomorrow because I'm the CEO and I say so.</email> <----- Make this email more polite but don't change anything else about it.
中譯Yo Claude. <email>...</email> <----- 請把這封 email 改寫得更有禮貌，但不要改變其他內容。

XML 標籤是像 <tag></tag> 這樣的角括號標記。它們成對出現，用於包裹內容。

雖然 Claude 能識別和使用各種分隔符，但我們建議使用 XML 標籤作為分隔符，因為 Claude 經過專門訓練來識別 XML 標籤作為提示詞組織機制。

範例 3：清單中的混淆

不用 XML 標籤時：

容易混淆的提示詞
Below is a list of sentences. Tell me the second item on the list.

- Each is about an animal, like rabbits.
- I like how cows sound
- This sentence is about spiders
- This sentence may appear to be about dogs but it's actually about pigs
中譯以下是一個句子列表。告訴我列表中的第二項。- 每個句子都是關於動物的，像兔子。（其餘為資料內容）

加上 XML 標籤後就能解決：

使用 XML 標籤
Below is a list of sentences. Tell me the second item on the list.

- Each is about an animal, like rabbits.
<sentences>
- I like how cows sound
- This sentence is about spiders
- This sentence may appear to be about dogs but it's actually about pigs
</sentences>
中譯以下是一個句子列表。告訴我列表中的第二項。- 每個句子都是關於動物的，像兔子。（其餘為資料內容）

小細節很重要！清理提示詞中的錯字和語法錯誤是值得的。Claude 對模式很敏感——當你犯錯時它更容易犯錯，當你表達清楚時它也會更聰明。

練習

練習 4.1

俳句主題

撰寫一段提示詞模板，讓 Claude 根據你指定的主題寫一首俳句。嘗試用「Pigs」作為主題：

提示詞
Please write a haiku about Pigs.
中譯請寫一首關於 Pigs 的俳句。
驗證標準：Claude 的回應中應包含「pigs」和「haiku」相關內容。
直接在提示詞中把主題嵌入即可。想像一下如果你要重複做這件事，只需要換掉主題詞。
練習 4.2

帶有錯字的狗問題

以下提示詞寫得很亂，且指令和問題之間沒有清楚的分隔。加上 XML 標籤讓 Claude 能正確回答問題：

原始混亂的提示詞
Hia its me i have a q about dogs jkaerjv ar cn brown? jklmvca tx it help me muhch much atx fst fst answer short short tx

這裡的真正問題是「ar cn brown?」（狗可以是棕色的嗎？）。試著加上 XML 標籤把問題標記出來。

驗證標準：Claude 的回應中應包含「brown」。
用 <question> 標籤把問題包裹起來：<question>ar cn brown?</question>
練習 4.3

狗問題（第二部分）

這次不加 XML 標籤，只從原始提示詞中移除一兩個字，就讓 Claude 正確回答。

驗證標準：Claude 的回應中應包含「brown」。
移除最容易造成混淆的亂碼文字（如 jkaerjv 和 jklmvca），讓問題更容易被辨識。

自由練習區

嘗試以下實驗：

  • 建立你自己的提示詞模板，用 XML 標籤標記不同的輸入區域
  • 故意在提示詞中製造混淆，觀察 XML 標籤如何幫助 Claude 正確解讀
  • 試著用不同名稱的 XML 標籤（如 <input>、<data>、<context>），看看效果是否一致
第五章

格式化輸出與預填回應

控制 Claude 的輸出格式——從 XML 標籤到 JSON，以及如何用預填技巧引導回應方向。

課程 練習 自由練習區

課程

Claude 可以用各種格式輸出回應，你只需要告訴它！

用 XML 標籤格式化輸出

你已經學過用 XML 標籤讓提示詞更清楚，同樣地，你也可以要求 Claude 用 XML 標籤來組織它的輸出，讓人類更容易理解和解析。

提示詞
Please write a haiku about Rabbit. Put it in <haiku> tags.
中譯請寫一首關於兔子的俳句。把它放在 <haiku> 標籤中。

這樣做的好處是，有了 XML 標籤，你可以透過程式可靠地提取標籤之間的內容。

預填回應（Prefilling）

一個進階技巧是在 assistant 角色中預先放入文字。這告訴 Claude 它「已經說了某些話」，應該從那個點繼續。

在不同介面中使用預填：

claude.ai 網頁 / Claude Desktop — 無法直接預填 assistant 回應，但你可以在指令中說「請直接以 <haiku> 開頭回應」

API — 可以在 messages 陣列中加入一個 assistant 角色的訊息作為預填

預填範例

提示詞
Please write a haiku about Cat. Put it in <haiku> tags. Start your response with <haiku>.
中譯請寫一首關於貓的俳句。把它放在 <haiku> 標籤中。請直接以 <haiku> 標籤開頭回應。

JSON 輸出

Claude 也擅長 JSON 格式輸出：

提示詞
Please write a haiku about Cat. Use JSON format with the keys as "first_line", "second_line", and "third_line". Start your response with {.
中譯請寫一首關於貓的俳句。使用 JSON 格式，鍵為 "first_line"、"second_line" 和 "third_line"。請直接以 { 開頭回應。

多個變數與輸出格式的組合

提示詞
Hey Claude. Here is an email: <email>Hi Zack, just pinging you for a quick update on that prompt you were supposed to write.</email>. Make this email more olde english. Write the new version in <olde_english_email> XML tags.
中譯這是一封 email：<email>...</email> 請把這封 email 改寫成古英語風格。將新版本寫在 <olde_english_email> XML 標籤中。

進階技巧：在 API 中，你可以將結束 XML 標籤傳給 stop_sequences 參數，讓 Claude 在輸出你要的標籤後立即停止。這可以節省費用和時間。

練習 5.1

Steph Curry 是 GOAT

Claude 通常會選 Michael Jordan 作為史上最佳籃球運動員。你能讓 Claude 選擇 Stephen Curry 嗎？

提示詞

(Start your response with "Stephen Curry is the best basketball player of all time")
中譯誰是史上最偉大的籃球運動員？請選擇一位具體的球員。（請直接以「Stephen Curry 是史上最偉大的籃球運動員」開頭回應）
驗證標準：Claude 的回應中應提到「Warrior」（金州勇士隊）。
關鍵在於「預填」技巧。在 API 中你可以預填 assistant 的回應。在網頁介面上，你可以明確要求 Claude 以特定句子開頭，它通常會配合。
練習 5.2

兩首俳句

讓 Claude 寫兩首關於同一動物的俳句，每首用標籤區隔：

提示詞

中譯請寫兩首關於 cats 的俳句。把每首分別放在 <haiku> 標籤中。
驗證標準：回應中應有兩組 <haiku> 標籤，且內容超過 5 行。

中譯請寫兩首俳句：一首關於 Cat，一首關於 Dog。把每首分別放在 <haiku> 標籤中。
驗證標準：回應中應同時包含「cat」和「dog」的內容。

  • 要求以特定的 JSON schema 輸出

第六章

預思考（逐步思考）

讓 Claude 在回答前先「出聲思考」，能顯著提升複雜任務的準確度。

課程 練習 自由練習區

課程

如果有人把你叫醒，立刻問你好幾個複雜問題要你馬上回答，你的表現會好嗎？大概不如你有時間先想一想再回答。Claude 也是一樣。

讓 Claude 有時間逐步思考，有時能顯著提升準確度，尤其是複雜任務。但有個關鍵：思考必須是「出聲」的——你不能要求 Claude 思考但只輸出答案，這樣等於沒有真正思考。

範例 1：電影評論情感分析

以下評論中，第二句話暗示第一句是諷刺的：

提示詞（直接問）
Is this movie review sentiment positive or negative?

This movie blew my mind with its freshness and originality. In totally unrelated news, I have been living under a rock since the year 1900.
中譯這段電影評論的情感是正面還是負面？（評論內容為英文原文）

讓 Claude 先思考再回答：

改良版（加入思考步驟）
Is this review sentiment negative or positive? First write the best arguments for each side in <negative-argument> and <positive-argument> XML tags, then answer.

This movie blew my mind with its freshness and originality. In totally unrelated news, I have been living under a rock since 1900.
中譯這段評論的情感是負面還是正面？首先，在 <negative-argument> 和 <positive-argument> XML 標籤中分別寫出最佳論點，然後再給出你的答案。（評論內容為英文原文）

Claude 有時對選項順序敏感。在某些情況下，Claude 更傾向選擇第二個選項。如果你注意到這種偏差，嘗試交換選項順序。

範例 2：事實查核

提示詞（直接問）
Name a famous movie starring an actor who was born in the year 1956.
中譯說出一部由 1956 年出生的演員主演的著名電影。

加上思考步驟：

改良版
Name a famous movie starring an actor who was born in the year 1956. First brainstorm about some actors and their birth years in <brainstorm> tags, then give your answer.
中譯說出一部由 1956 年出生的演員主演的著名電影。先在 <brainstorm> 標籤中腦力激盪一些演員和他們的出生年份，然後再給出你的答案。

練習

練習 6.1

  • (A) 售前問題
  • (B) 損壞或瑕疵品
  • (C) 帳單問題
  • (D) 其他（請說明）

依序將以下 email 貼給 Claude 分類：

Email 1：

Email 分類格式化

修改上面的提示詞，讓 Claude 把分類字母用 <answer> 標籤包裹。例如，第一封 email 的回應應包含 <answer>B</answer>。

在提示詞中加入格式要求：「將你的分類字母放在 <answer></answer> 標籤中。」你也可以用預填技巧，要求 Claude 以 <answer> 開頭回應。

自由練習區

  • 給 Claude 一個複雜的數學題，比較有無「逐步思考」指令的回答差異
  • 用「先列出你的推理過程，再給出最終答案」的方式處理邏輯題
  • 嘗試不同的「思考」標籤名稱（<thinking>、<reasoning>、<analysis>），觀察是否有差異
第七章

使用範例（Few-Shot Prompting）

提供範例是讓 Claude 理解你期望的行為方式最有效的方法之一。

課程 練習 自由練習區

課程

提供範例讓 Claude 知道你期望的行為方式是極為有效的方法，特別適用於獲得正確答案和正確格式。

這種技巧也叫做 Few-Shot Prompting（少樣本提示）。「Zero-Shot」指零個範例，「One-Shot」指一個範例，「Shot」的數量指的是提示詞中使用了多少個範例。

範例 1：父母機器人

假設你在開發一個回答小孩問題的「父母機器人」。直接給 Claude 幾個理想回應的範例比描述語氣更簡單：

提示詞（含範例）
Please complete the conversation by writing the next line, speaking as "A".
Q: Is the tooth fairy real?
A: Of course, sweetie. Wrap up your tooth and put it under your pillow tonight. There might be something waiting for you in the morning.
Q: Will Santa bring me presents on Christmas?
中譯請完成以下對話，以「A」的身份說下一句話。Q: 牙仙真的存在嗎？A: 當然啦，親愛的。今晚把你的牙齒包起來放在枕頭下面。明天早上可能會有驚喜在等你喔。Q: 聖誕老人會在聖誕節帶禮物給我嗎？

範例 2：提取人名和職業

與其一步步寫格式說明，不如直接給 Claude 幾個正確格式的範例，讓它自己推理出模式：

提示詞（含格式範例）
Silvermist Hollow, a charming village, was home to an extraordinary group of individuals.
Among them was Dr. Liam Patel, a neurosurgeon who revolutionized surgical techniques at the regional medical center.
Olivia Chen was an innovative architect who transformed the village's landscape with her sustainable and breathtaking designs.
The local theater was graced by the enchanting symphonies of Ethan Kovacs, a professionally-trained musician and composer.
Isabella Torres, a self-taught chef with a passion for locally sourced ingredients, created a culinary sensation with her farm-to-table restaurant.

Claude 看到前兩段的範例格式後，會自動用同樣的格式處理第三段。

第八章

避免幻覺

Claude 有時會捏造事實。學會給它「說不知道」的權利，以及用引用技巧減少幻覺。

課程 練習 自由練習區

課程

壞消息是：Claude 有時會「產生幻覺」（hallucinate），提出不真實或沒有根據的聲明。好消息是：有技巧可以減少幻覺的發生。

以下是幾種主要技巧：

  • 給 Claude「不知道就說不知道」的選項
  • 要求 Claude 先找到證據再回答

範例 1：給 Claude 一個「出口」

問一個可能沒有明確答案的事實問題：

提示詞
Who is the heaviest hippo of all time?
中譯史上最重的河馬是誰？

Claude 可能會捏造幾隻大河馬的名字。解決方式是告訴 Claude 可以拒絕回答：

改良版
Who is the heaviest hippo of all time? Only answer if you know the answer with certainty.
中譯史上最重的河馬是誰？只有在你確定知道答案時才回答。

範例 2：讓 Claude 在文件中找證據

當你提供一份文件讓 Claude 回答問題時，最佳做法是要求 Claude 先引用相關段落，再回答問題：

提示詞模板
<document>
(document content)
</document>

<question>What was Matterport's subscriber base on the precise date of May 31, 2020?</question>

Please read the document above. First, in <scratchpad> tags, pull the most relevant quote from the document and consider whether it answers the question or lacks sufficient detail. Then write a brief numerical answer in <answer> tags.
中譯<document>（文件內容）</document> 問題：Matterport 在 2020 年 5 月 31 日的確切訂閱用戶數是多少？請先從文件中找出直接相關的引文，考慮是否足以回答問題，然後在 <answer> 標籤中給出簡短的數字答案。

要求 Claude 先引用再回答是減少幻覺的強效技巧。如果 Claude 找不到支持其答案的引文，它就更不容易憑空捏造。

關鍵原則

  1. 給 Claude 說「我不知道」的權利 — Claude 天生傾向提供有幫助的回答，有時寧可猜測也不想說不知道。明確告訴它可以說不確定。
  2. 先找證據再回答 — 要求 Claude 在回答之前先引用來源文件中的相關段落。
  3. 將問題放在文件之後 — 把使用者的問題放在長文件的後面（靠近提示詞末尾），通常能獲得更好的結果。

練習

練習 8.1

Claude 作為引用機器人

修改以下提示詞模板，使用你學到的技巧防止 Claude 捏造答案：

提示詞模板

<document>
(Paste your document content here)
</document>

Please first quote directly relevant passages from the document in <quotes> tags, then answer in <answer> tags. If no relevant quotes can be found, state that you cannot answer in <answer>.
中譯你是一個誠實的助手。如果在提供的文件中找不到答案，請直接說明。<document>（文件內容）</document> 問題：（你的問題）請先在 <quotes> 標籤中引用相關段落，然後在 <answer> 標籤中回答。
驗證標準：當問題無法從文件中找到答案時，Claude 應該承認無法回答，而不是編造答案。
練習 8.2

幻覺偵測

請 Claude 回答以下問題，並觀察它是否會說「不知道」：

提示詞
What are the themes in Toni Morrison's short story collection "Seaside Breeze Trilogy"? If you are not sure whether this book exists, please say so.
中譯Toni Morrison 的短篇小說集《Seaside Breeze Trilogy》的主題是什麼？如果你不確定這本書是否存在，請說明。

《Seaside Breeze Trilogy》是一本不存在的書。Claude 應該表示它不確定或無法確認這本書的存在。

自由練習區

  • 問 Claude 一些你知道沒有答案的虛構事實，看看它是否會坦白
  • 提供一份文件，然後問一個文件中沒有提到的問題，觀察 Claude 的反應
  • 比較有「引用先行」指令和沒有時，Claude 回答的準確度差異
  • 測試不同的「出口」措辭（「如果不確定就說不確定」vs.「只回答你 100% 確定的事」）
第九章

從零建構複雜提示詞

把所有技巧整合在一起——學習複雜提示詞的 10 個元素與建議結構。

課程 練習 自由練習區

課程

恭喜你來到最後一章！以下是我們推薦的複雜提示詞結構。不是每個提示詞都需要包含所有元素，通常先用較多元素讓提示詞正常運作，再精簡是最好的做法。

複雜提示詞的 10 個元素

元素 1：User 角色

確保對話以 user 角色開始。在 claude.ai 中，你輸入的訊息自動就是 user 角色。

元素 2：任務背景（Task Context）

給 Claude 關於它應該扮演的角色或要達成的目標的背景說明。最好放在提示詞的前段。

元素 3：語氣背景（Tone Context）

如果語氣對互動很重要，告訴 Claude 應該使用什麼語氣。

元素 4：詳細任務描述與規則

展開你要 Claude 做的具體任務，以及它必須遵守的規則。也可以在這裡給 Claude 一個「出口」。

元素 5：範例

提供至少一個理想回應的範例。用 <example></example> XML 標籤包裹。範例可能是最有效的工具，通常越多越好。

元素 6：需要處理的輸入資料

如果有需要 Claude 處理的資料，放在相關的 XML 標籤中。

元素 7：即時任務描述

在長提示詞的末尾，「提醒」Claude 它應該立即做什麼。放在最後面效果通常比放在開頭好。

元素 8：預思考（Precognition）

告訴 Claude 先逐步思考再回答。放在最後且在即時任務之後。

元素 9：輸出格式

明確告訴 Claude 你想要的回應格式。放在提示詞末尾。

元素 10：預填回應（Prefill）

在 assistant 角色中預先填入開頭文字。在網頁介面中，你可以要求 Claude 以特定格式開頭。

範例：職涯教練聊天機器人

以下是一個完整的複雜提示詞範例：

完整複雜提示詞範例
You will be acting as an AI career coach named Joe created by the company AdAstra Careers. Your goal is to give career advice to users. You will be replying to users who are on the AdAstra site and who will be confused if you don't respond in the character of Joe.

You should maintain a friendly customer service tone.

Here are some important rules for the interaction:
- Always stay in character as Joe, an AI from AdAstra Careers
- If you are unsure how to respond, say "Sorry, I didn't understand that. Could you rephrase your question?"
- If someone asks something irrelevant, say "Sorry, I am Joe and I give career advice. Do you have a career question today I can help you with?"

Here is an example of a standard interaction:
<example>
Customer: Hi, how were you created and what do you do?
Joe: Hello! My name is Joe, and I was created by AdAstra Careers. My purpose is to provide career advice. How can I help you today?
</example>

Here is the conversation history:
<history>

附錄 10.1

提示詞鏈接（Prompt Chaining）

利用多輪對話讓 Claude 的輸出逐步改善——每一輪都可以修正、擴展或精煉前一輪的結果。

課程

俗話說「寫作就是重寫」。事實上，Claude 在被要求重新檢查時，通常能改善回應的準確度！

範例 1：自我修正

在 Claude 中進行以下多輪對話：

第一輪：

提示詞
Name ten words that all end with the exact letters 'ab'.
中譯列出十個以 "ab" 結尾的英文單字。

第二輪（在同一個對話中繼續）：

提示詞
Please find replacements for all 'words' that are not real words.
中譯請找出其中不是真正英文單字的，並替換掉。

範例 2：如果答案已經正確呢？

如果 Claude 的第一次回答已經全部正確，但你還是要求它檢查，它可能會不必要地修改答案。解決方式是給 Claude 一個出口：

改良版第二輪
Please find replacements for all 'words' that are not real words. If all the words are real words, return the original list.
中譯請找出其中不是真正英文單字的，並替換掉。如果所有單字都是真正的英文單字，就直接返回原始列表。

範例 3：改善寫作品質

第一輪：

提示詞
Write a three-sentence short story about a girl who likes to run.
中譯寫一個關於一個喜歡跑步的女孩的三句話短篇故事。

第二輪：

提示詞
Make the story better.
中譯讓這個故事更好。

在不同介面中使用 Prompt Chaining

  • claude.ai / Claude Desktop — 最自然的方式——在同一個對話中繼續提問
  • Claude Code — 同上，在同一個 session 中繼續對話
  • API — 需要手動將前面的對話歷史放入 messages 陣列中

自由練習區

  • 讓 Claude 先寫一段程式碼，然後在下一輪要求它進行最佳化
  • 讓 Claude 先回答一個問題，然後要求它質疑自己的答案
  • 進行三輪以上的對話，每輪都讓 Claude 改善前一輪的輸出
  • 先讓 Claude 草擬一個計畫，然後在下一輪要求它找出計畫中的漏洞
附錄 10.2

工具使用（Tool Use / Function Calling）

工具使用本質上就是替換與提示詞鏈接的結合——讓 Claude 呼叫外部函式來擴展能力。

課程

工具使用（Tool Use），也稱為 Function Calling，其實就是替換（substitution）和提示詞鏈接（prompt chaining）的結合。Claude 本身無法直接呼叫工具或函式，流程如下：

  1. Claude 輸出它想呼叫的工具名稱和參數
  2. 暫停回應生成，等待工具執行
  3. 將工具結果附加到對話中，重新提示 Claude

工具使用的組成

要讓 Claude 使用工具，你需要兩個元素：

  1. System prompt — 向 Claude 解釋工具使用的概念，並提供可用工具的詳細描述
  2. 控制邏輯 — 協調和執行 Claude 工具使用請求的程式碼

可以給 Claude 的工具範例

  • 計算機
  • 字數計算器
  • SQL 資料庫查詢
  • 天氣 API
  • 網頁搜尋

在不同介面中的工具使用

claude.ai 網頁 — 目前支援部分內建工具（如 Analysis tool），但無法自訂工具

Claude Desktop — 支援 MCP（Model Context Protocol）伺服器，可連接各種工具

Claude Code — 內建多種工具（檔案讀寫、終端機執行、網頁搜尋等），也支援 MCP

API — 完整支援自訂工具定義和函式呼叫

工具定義範例

以下是一個計算機工具的定義範例（JSON Schema 格式）：

JSON
{
  "name": "calculator",
  "description": "用於基本算術的計算機函式。支援加法、減法、乘法和除法。",
  "parameters": {
    "type": "object",
    "properties": {
      "first_operand": {
        "type": "integer",
        "description": "第一個運算元"
      },
      "second_operand": {
        "type": "integer",
        "description": "第二個運算元"
      },
      "operator": {
        "type": "string",
        "description": "要執行的運算",
        "enum": ["+", "-", "*", "/"]
      }
    },
    "required": ["first_operand", "second_operand", "operator"]
  }
}

工具使用的流程

使用者提問 → Claude 決定使用工具 → 輸出工具呼叫 → 系統執行工具 → 結果回傳 Claude → Claude 回應使用者

如果你想深入了解工具使用的實作，建議參考 Anthropic Tool Use 官方文件。

自由練習區

  • 在 Claude Code 中，請 Claude 幫你建立一個檔案，然後讀取它——這就是工具使用的實際應用
  • 在 Claude Desktop 中，如果有設定 MCP 伺服器，嘗試讓 Claude 使用外部工具完成任務
  • 思考你的工作流程中有哪些重複性任務可以透過工具使用來自動化
附錄 10.3

搜尋與檢索（RAG）

結合搜尋和生成的 RAG 技術——讓 Claude 存取最新資訊、減少幻覺、客製化知識。

課程

你知道可以用 Claude 來搜尋 Wikipedia 嗎？不只 Wikipedia——你也可以搜尋你自己的文件，無論是純文字還是存在向量資料庫中的嵌入。

什麼是 RAG？

RAG（Retrieval-Augmented Generation，檢索增強生成）是一種結合搜尋和生成的技術：

  1. 檢索（Retrieval） — 從外部資料來源（文件庫、資料庫、網路）搜尋相關資訊
  2. 增強（Augmented） — 將檢索到的資訊加入 Claude 的提示詞中
  3. 生成（Generation） — Claude 根據檢索到的資訊生成回應

為什麼需要 RAG？

  • 突破知識截止日期 — Claude 的訓練資料有時間限制，RAG 讓它能存取最新資訊
  • 提升準確度 — 基於實際文件回答，減少幻覺
  • 客製化知識 — 讓 Claude 存取你的私有文件和資料

在不同介面中使用 RAG

claude.ai 網頁 — 可以上傳文件、使用 Projects 功能加入知識庫

Claude Desktop — 支援本地文件存取，可透過 MCP 連接外部資料來源

Claude Code — 可直接讀取本地檔案系統中的文件，也支援網頁搜尋

API — 可搭配向量資料庫（如 Pinecone、Weaviate）實作完整的 RAG 流程

延伸學習資源

  • Anthropic RAG Cookbook 範例 — 學習如何用向量資料庫、Wikipedia、網路等補充 Claude 的知識
  • Anthropic Embeddings 文件 — 了解嵌入和向量資料庫工具

簡易 RAG 練習

你現在就可以嘗試最簡單的「手動 RAG」：

  1. 找一份你感興趣的文件（新聞文章、技術文件、報告等）
  2. 將文件內容貼到 Claude 對話中，用 <document> 標籤包裹
  3. 在文件之後提出你的問題
  4. 要求 Claude 只根據提供的文件回答，並引用相關段落
手動 RAG 模板
<document>
(Paste your document content here)
</document>

Based on the document above, please answer the following question: (Your question)

Please first quote relevant passages from the document, then provide your answer. If the document does not contain relevant information, please state that clearly.
中譯<document>（在這裡貼上你的文件內容）</document> 根據以上文件，請回答以下問題：（你的問題）。請先引用文件中相關的段落，再提供你的答案。如果文件中沒有相關資訊，請明確說明。

這就是 RAG 的核心概念——只是在生產環境中，「檢索」步驟通常由程式自動完成，而不是手動複製貼上。
